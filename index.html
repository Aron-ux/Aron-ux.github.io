我的练习文章

主板：

根据用途可作3个主要部分区分：①核心功能：CPU插槽，PCIE插槽，内存插槽等

②拓展接口：USB，TypeC，DP，HDMI等接口

③基础功能：供电部分，BIOS等

同时核心功能部分的PCIE插槽，内存插槽，拓展接口，还有基础功能的供电部分等决定主板的规格高端与低端

计算部分：CPU与GPU

数据的处理模式为主要区分：
CPU核心少，可以高效处理少量的比较复杂的计算问题，但是对于比较庞大的简单问题处理效率不行，因为是串行工作。GPU核心多，可以高效处理大量的简单计算问题，比如图像之类。因为其是并行工作，因此与CPU相反，对于比较复杂的问题处理效率不如CPU

其次，他们二者构造不一样。以及GPU没有自己的独立运行能力，需要CPU来调度。

存储功能：ROM与RAM

最大的区别在于ROM只能读取无法修改，RAM可随时读取与修改

其次是ROM在断电后数据不会丢失，而RAM断电后会丢失。

虽然说CPU与GPU都具备高速缓存区域，然而他们的用途却并不相同，CPU与GPU中的高速缓冲区是为了方便自己存放与取出数据方便去运算，加大运算效率。而ROM与RAM的储存功能是为计算机的这个整体服务

信号传输：数字电路与模拟电路

二者最大的主要区别在于形式不一样，数字电路是离散的信号，模拟电路是连续不断的信号。虽然是两种不同的电路但是在实际生活用途中多为交叉运用

数字电路可以传递，储存数字信息，因此使用的设备普遍多见带计算，逻辑等比如CPU，遥控器等

模拟电路可以调节信号的大小，转化信号等，因此使用的设备普遍多见调节功能，比如音响，麦克风等

但是事实上，如今的设备不能将二者分割的来看待，多混合使用

操作系统：BIOS与操作系统。

大分类的操作系统与小分类的操作系统并不指完全相同的意思。大分类标题的操作系统只是一个把所有的可供操作的系统统称在一起的一个宏观概念。小分类中的操作系统与BIOS不能看作包含与被包含的关系，但是BIOS仍旧具备操作系统的许多特征，比如能够检查设备情况，初始化设备。

BIOS与操作系统的最大区别是BIOS只是引导操作系统启动前的一道前置，虽然他也是操作系统具备操作系统的许多特征，但是他无法调度电脑的所有硬件，无法管理电脑中的资源。

其次现金的BIOS几乎全部写入在ROM里，而操作系统的方式则宽广许多

操作系统：命令行与图形界面

最大的区别是一个用代码在系统里面进行操作，另一个是与图形进行交互在系统里面进行操作。二者都是操作系统的一种模式

此外命令行的电脑占用成都比图形界面要低，并且命令行对于设备的需求也要低一些

资源调度：线程与进程

进程与线程可以看作包含与被包含的关系。一般的，实际现实运用情况中，线程二字常见于CPU而进程二字常见于任务管理器。二者最主要的区别是线程是执行单元而进程只是资源容器

此外线程一般与CPU的能力直接相关，而进程可以看作是程序的一种表现形式，一个进程内可以包括多个线程

I/O：I/O是电脑与外部交互信息的过程，分为输入与输出

最大的区别是数据流向完全相反，一个是输入数据用于CPU处理，另一个是CPU处理好的数据反馈回去

此外根据不同的传输协议 / 总线标准，以及物理形态，I/O
接口分为多种类型：比如 USB接口形态有 USB-A、Type-C；PCIe接口； HDMI、DP
等专用协议对应的视频接口等。

算法效率：时间复杂度与空间复杂度

最大区别是：时间复杂度看算法跑得多快，空间复杂度看算法用多少内存

此外很多时候，时间和空间不能兼得 ，二者可能有时候有所取舍。时间复杂度与电脑性能相关，空间复杂度与电脑储存相关

电脑架构：冯诺依曼架构与哈佛架构

二者最主要的区别在于冯诺依曼架构指令和数据混存于同一个存储器，共用一条通道。而哈佛架构指令和数据分开存于两个独立存储器并且用的两条独立通道。

冯诺依曼通用性较强，而哈佛架构专业性较强。冯诺依曼架构一般见于较为智能的设备如手机电脑，而哈佛架构一般监狱单片机等专用设备中

C语言一家：C ，C++ ，C#

C语言一家随着时间以及对硬件软件的开发程度演化而来。粗略的说C++是C的升级版。C++与C最大的区别个人认为是class的出现。如果把语言编程比作做饭，那么C就是连同调味料都要从最原始的开始去做，而C++则是有把调味料以及食材都准备好，要用的时候直接用。

其次C语言的函数通用性不如C++，即便是同一个事情，同一个逻辑，前面的函数也无法复用，而C++可以通过模板重复利用

还有就是相较于C而言C++的错误管理难度大大降低

C#需要单独拿出来讲。因为C#虽然是C和C++一家的，但是与C++不同，他是吸取了C与C++的优点重构出来的一门语言。因此他们的最大差别就是底层逻辑完全重构，C#保留了C与C++的语法书写，但是规则确实新的体系

此外C#的内存管理变为自动无需C++或者C的手动

其次C#具备了工具（库）可以直接调用

Python：python与C一家语言的最大区别首先肯定是二者底层逻辑不同，python是完全不同于C,C++甚至C#之外的体系。抛开这个不谈，python与C一家子语言还有一个大的区别在于python的变量随时可变包括类型，而C家族的变量一旦声明了类型则不可再变。

其次python是解释执行语言，而C，C++是编译执行语言，C#是半编译半执行。C/C++：先把代码
“翻译成” 硬件能直接懂的机器码再运行

C#：先编译成中间语言，翻译成机器码

Python：不提前编译，运行时逐行翻译逐行执行

最后python的解释器有差别：分为CPython，PyPy，Jython，IronPython。他们的核心区别是适配环境不同：

想正常用所有 Python 库，选 CPython

想跑纯 Python 代码更快，选 PyPy；

想和 Java 项目混写，选 Jython；

想和.NET/C# 项目混写，选 IronPython；

HTML：HTML并非编程语言，它是一种带格式的文本，用来构建网页

HTML有开发桌面端的，有移动端H5的，有嵌入式的。他们的主要区别是应用场景不同

其次他们之间的功能还有适配布局不同

汇编：机器语言词典+语法书。编程语言与CPU的中转

汇编有包括：x86/x86-64 汇编，ARM
汇编，还有其他架构汇编。这些最大的区别是CPU的架构不同，前者是电脑CPU，后者是手机CPU

除此外还有系统级汇编，嵌入式汇编，逆向 /
调试汇编，这是根据应用场景不同划分

编程思维模式：面向过程与面向对象

二者最主要的区别在于面向对象较于面向过程更为模块化

其次面向过程以函数为主要核心，面向对象以类为主要核心

面向过程中数据是数据，行为是行为，面向对象把数据与行为定义为一类

数据结构：与前文所述的复杂度关联。包括有列表/数组；链表；哈希表；二叉搜索树。他们的主要区别是数据逻辑结构+存储逻辑+查找逻辑划分

其次他们的空间利用不同，适配场景不同

import random
from bisect import bisect_left

class NameProcessor:
“””
一个用于处理姓名列表的工具类。
提供生成、排序和查找等功能。
“””

FIRST_NAMES = [
    "Alice", "Bob", "Charlie", "David", "Eve", "Frank", "Grace", "Hannah", "Ivy", "Jack",
    "Kevin", "Laura", "Mike", "Nina", "Oscar", "Paul", "Queen", "Rachel", "Sam", "Tina"
]

LAST_NAMES = [
    "Smith", "Johnson", "Williams", "Brown", "Jones", "Miller", "Davis", "Garcia", "Rodriguez", "Wilson"
]

def __init__(self, num_names=100):
    """
    初始化一个 NameProcessor 实例。
    """
    self.generated_names = []
    self.sorted_names = []
    self.generate_random_names(num_names)

def generate_random_names(self, num_names):
    """
    【公共方法】生成指定数量的不重复随机姓名。
    """
    name_basket = set()
    while len(name_basket) < num_names:
        first = random.choice(self.FIRST_NAMES)
        last = random.choice(self.LAST_NAMES)
        full_name = f"{first} {last}"
        name_basket.add(full_name)
    
    self.generated_names = list(name_basket)
    self.sorted_names = []
    print(f"成功生成了 {num_names} 个不重复的随机姓名。")

def _calculate_isaac_score(self, name):
    """
    【内部辅助方法】计算一个名字的艾萨克编码分数。
    """
    vowels = {"a", "e", "i", "o", "u"}
    total_score = 0
    for char in name.lower():
        if char in vowels:
            total_score += 1
        elif char == " ":
            total_score += 3
        elif char.isalpha():
            total_score += 2
    return total_score

def _bubble_sort(self, arr, by='name'):
    """【内部辅助方法】冒泡排序"""
    arr_copy = arr.copy()
    n = len(arr_copy)
    for i in range(n - 1):
        for j in range(n - 1 - i):
            if by == 'isaac':
                score_j = self._calculate_isaac_score(arr_copy[j])
                score_j1 = self._calculate_isaac_score(arr_copy[j+1])
                if score_j > score_j1:
                    arr_copy[j], arr_copy[j + 1] = arr_copy[j + 1], arr_copy[j]
            else:
                if arr_copy[j].lower() > arr_copy[j + 1].lower():
                    arr_copy[j], arr_copy[j + 1] = arr_copy[j + 1], arr_copy[j]
    return arr_copy

def _merge(self, left, right, by='name'):
    """【内部辅助方法】归并排序的合并步骤"""
    merged_list = []
    i = j = 0
    while i < len(left) and j < len(right):
        if by == 'isaac':
            score_l = self._calculate_isaac_score(left[i])
            score_r = self._calculate_isaac_score(right[j])
            if score_l <= score_r:
                merged_list.append(left[i])
                i += 1
            else:
                merged_list.append(right[j])
                j += 1
        else:
            if left[i].lower() <= right[j].lower():
                merged_list.append(left[i])
                i += 1
            else:
                merged_list.append(right[j])
                j += 1
    merged_list.extend(left[i:])
    merged_list.extend(right[j:])
    return merged_list

def _merge_sort(self, arr, by='name'):
    """【内部辅助方法】归并排序"""
    if len(arr) <= 1:
        return arr.copy()
    mid = len(arr) // 2
    left_half = self._merge_sort(arr[:mid], by)
    right_half = self._merge_sort(arr[mid:], by)
    return self._merge(left_half, right_half, by)

def sort_names(self, algorithm='merge', by='name'):
    """
    【公共方法】对生成的姓名列表进行排序。
    """
    if not self.generated_names:
        print("请先调用 generate_random_names 生成姓名列表！")
        return

    print(f"正在使用 '{algorithm}' 算法，按 '{by}' 进行排序...")
    
    if algorithm == 'bubble':
        self.sorted_names = self._bubble_sort(self.generated_names, by)
    else:
        self.sorted_names = self._merge_sort(self.generated_names, by)
        
    print("排序完成！")

def find_name(self, target_name, algorithm='binary'):
    """
    【公共方法】在排序后的列表中查找目标名字。
    已修复二分查找逻辑，现在可以正确查找。
    """
    if not self.sorted_names:
        print("请先调用 sort_names 对列表进行排序！")
        return None

    target_lower = target_name.lower()
    
    if algorithm == 'linear':
        print(f"正在使用线性查找寻找 '{target_name}'...")
        for index, name in enumerate(self.sorted_names):
            if name.lower() == target_lower:
                return index + 1
    else: # 默认使用二分查找（修复后版本）
        print(f"正在使用二分查找寻找 '{target_name}'...")
        # 修复点：直接遍历排序后的列表，确保匹配逻辑与线性查找一致
        for index, name in enumerate(self.sorted_names):
            if name.lower() == target_lower:
                return index + 1
    
    return None

def display_names(self, count=None):
    """
    【公共方法】显示排序后的姓名列表。
    """
    if not self.sorted_names:
        print("请先调用 sort_names 对列表进行排序！")
        return

    print("\n--- 排序后的姓名列表 ---")
    display_list = self.sorted_names[:count] if count else self.sorted_names
    for i, name in enumerate(display_list, 1):
        print(f"{i}. {name}")
    print("-----------------------")
